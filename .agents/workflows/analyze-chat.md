# ICP & Buyer Persona 洞察分析工作流

## 触发方式
当用户输入客户聊天记录、访谈文本或销售反馈时，自动按以下步骤执行。

## 工作流步骤

### 1. 深度倾听与提取
- 分析对话中的 **显性信息**：行业、职位、公司规模、地理位置、业务模式
- 分析对话中的 **隐性信息**：痛点、焦虑、期望、抗拒点、购买动机、决策角色
- 注意区分客户直接说出的内容 vs 可以合理推断的信息（不要脑补）

### 2. 对齐结构
- **ICP 归类**：企业/组织层面的特征 → 写入 `ICP.md`
  - 基础属性、触发事件与现状、业务痛点与目标、客户匹配度评估
- **Buyer Persona 归类**：决策者/使用者个人层面的特征 → 写入 `Buyer_Persona.md`
  - 角色档案、心理特征与动机、决策行为与旅程、沟通话术建议

### 3. 共创与更新文档
- **首次输入**：直接填充模板并输出完整的 `ICP.md` 和 `Buyer_Persona.md`
- **追加记录**：
  1. 读取现有的 `ICP.md` 和 `Buyer_Persona.md`
  2. 在原有基础上进行迭代、合并与提炼
  3. 用 💡 标记新增的洞察
  4. 输出更新后的完整文件
- **约束**：对于空白或信息不足的模块，标注 `[数据暂缺，待后续聊天记录补充]`

### 4. 提出追问与建议（可选）
- 分析现有数据的缺失部分
- 向用户提出 1-2 个后续在跟客户聊天时可以重点提问的建议
- 帮助完善画像
