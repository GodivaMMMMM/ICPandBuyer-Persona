# **产品概览（IP Transit）**

- 许多企业需要全球一致的数字体验，但缺乏资源和专业能力。
- **Zenlayer IP Transit** 提供“一键即开”的全球带宽连接，快速高效。

- **接入优质网络**：可自动接入 Zenlayer 全球 PoP 的高端混合网络，或手动选择运营商与路由。

- **卓越服务：**
    - IP Transit 提供 **99.95% SLA**，电路与硬件内置冗余。
    - 提供 **全天候 7×24 技术支持**，行业领先的 **15 分钟内响应**。
    - Zenlayer 在 **4 小时内解决 95% 的工单**。

# **产品功能（Product Features）**

## **冗余与弹性连接（**Redundant and resilient connectivity**）**

IP Transit 帮助企业在任意区域构建超可靠网络，提供全冗余架设计，消除单点故障。通过网络多样性进一步提升弹性，包括城域网与长途网络互联。

- **网络结构冗余：**IP Transit 配置 **2 台路由器 + 2 台交换机，**避免设备硬件故障引起的宕机。
- **上游冗余：**Zenlayer 提供 **2 个以上 ISP 连接**，避免单一 ISP 故障，确保完整冗余。
- **路由协议冗余：**IP Transit 支持 **BGP（边界网关协议）**，实时调整线路，进一步增强网络稳定性。

## **全网互联设计（Full mesh design）**

Zenlayer 采用全网互联设计确保高可用性。所有节点分为 **核心节点** 和 **边缘节点**：

- 与各大主流内容平台：Google、Amazon、TikTok、Meta、Apple 等建立高效互联
- 与 Telefonica、Etisalat、Telkom 等全球优质 ISP 建立直连
- 1.2 Tbps 国际骨干带宽，4T 本地互联带宽
- 为保障网络稳定，每个节点均接入 **两条以上完整路由 ISP**

## **多层级监控（Multi-level monitoring）**

- 全球 550+ 网络监测点部署自动告警机制
- 动态路由优化，自动选择最优传输路径，显著减少网络跳数与传输延迟
- 基于 BGP 的客户流量工程策略
- 高冗余连接 + 全天候监控，确保全球范围内的超可靠网络

## **前缀与开放对等策略**

- 支持 **4000+ IPv4** 和 **50+ IPv6** 前缀（AS21859）
- 采用开放对等策略（Open Peering Policy），与本地运营商互联，混合 BGP 提升性能
- 支持购买地理位置 IP，用于战略性部署和更优的区域性能
- 支持全球范围的 ASN 与 IP 前缀公告

# Zenlayer 运营的两个自治系统号（ASN）

- **AS21859**
AS21859 是高性能本地用户网络（eyeball network），分布式部署于多个城市。该网络与多家本地互联网服务提供商（ISP）建立了广泛的对等互联，从而有效提升用户访问流量与本地内容的网络性能。
- **AS4229**
AS4229 为优质内容网络。该网络与内容提供商、互联网上游供应商以及各大互联网交换中心建立了广泛的对等互联，并具备骨干网络能力，可高效实现 Zenlayer 边缘节点在多个城市间的互联互通。借由这一网络，ISP 客户能够有效降低成本、改善内容访问体验，并显著提升上行网络性能。

# IPT的网络配置

| 接入模式        | 场景说明                                                     | 关键点                                                       | 适用客户                                                     |
| --------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 网关模式        | * 客户无需自己建立路由，只需将流量交给 Zenlayer。 * Zenlayer 提供公网地址段，并在 **BR（Border Router）边界路由器** 上配置网关。 * 客户的设备（CE）直接使用该网关访问互联网。 | * 连接使用 **公网地址**。 * 不需要运行路由协议（无静态/动态路由）。 * Zenlayer 负责网络出口转发。客户只需购买 Zenlayer 的公网地址段。 | 网络简单、没有自有 ASN，仅需要快速获得互联网访问的客户（如中小企业、托管用户）。 |
| 静态路由模式    | * 客户希望对流量路径有一定控制能力，但不运行复杂协议。 * CE 与 BR 之间建立 **静态路由**，用私网地址互联。 * 公网路由信息通过静态路由传递给 BR。 | * CE ↔ BR 之间使用 **私网地址**。 * 使用静态路由方式传递公网地址段（如 124.0.0.0/24）。 * 客户仍需向 Zenlayer 购买公网地址段。 * Console 已支持自动化配置。 | 需要简单的冗余控制和固定路径，但不具备 ASN/BGP 能力的客户。  |
| BGP动态路由模式 | * 客户有自己的 ASN，希望控制路由策略（多出口、优选路径等）。 * CE 与 BR 之间建立 **BGP 对等关系**，动态交换路由。 * 客户的公网路由由自身 ASN 宣告，Zenlayer 负责全网传播。 | * 使用 **BGP 协议** 建立动态路由关系。 * 客户可带自有 ASN 和 IP。 * 无需购买 Zenlayer 公网地址。 * 可实现智能路径、自动 Failover、多出口负载均衡。 | 有网络运营能力的 ISP、CDN、大型 SaaS 或跨国企业。            |
