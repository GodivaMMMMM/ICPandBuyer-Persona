# 架构图

**Zenlayer 云网络（Cloud Networking）** 的整体连接拓扑：

它以 **Zenlayer 全球骨干网** 为核心，通过多种接入方式将 **办公场所、数据中心、公有云、移动用户** 等不同节点高效互联，实现 **混合云、多云、全球企业网络、SaaS 优化** 等应用场景。

## 核心组成部分

### **1️⃣ Zenlayer 骨干网**

- 全球分布式节点互联，构成高可靠、低延迟的私有骨干网络。
- 提供端到端的智能路由与带宽调度能力。
- 支持灵活的带宽扩展、QoS 优化和弹性配置。

**2️⃣ 连接端类型**

| 类型 | 描述 | 接入方式 |
| --- | --- | --- |
| **办公室（总部、分支、门店）** | 用于企业总部与各地分支机构互联 | LL专线 / IPSec 加密隧道 |
| **数据中心（机房/托管、BMC/VM）** | 连接企业自建或托管设备及虚拟机资源 | LL / XC 专线 或 IPSec 接入 |
| **公有云（AWS、Azure、AliCloud、GCP、UCloud等）** | 与云厂商私网高速互通 | DX（Direct Connect）专线 |
| **移动用户** | 支持员工或远程办公终端的安全访问 | SSL / IPSec VPN（如 Fortinet Client 或 OpenVPN） |

## **方案类型**

- **二层方案（L2）**：点到点直连，适用于数据中心互联或专线备份。
- **三层方案（L3）**：点到多点网络互通，适用于多节点、跨区域组网。

## **接入类型**

- **公有云**：通过 DX（Direct Connect）直连。
- **办公室**：LL 专线 或 IPSec VPN。
- **数据中心**：LL/XC 专线 或 IPSec VPN（同 POP 建 XC）。
- **算力（BMC/VM/ZEC）**：多数节点直接接入 Zenlayer 骨干网。
- **移动用户**：通过 Fortinet Client / OpenVPN 接入。

## **网关类型（Gateway Types）**

| 类型 | 支持能力 |
| --- | --- |
| OFS | 支持LL/XC/DC的二层方案 |
| PE | 支持LL/XC/DC的二层与三层方案 |
| **Fortinet vPOP / MCE** | 支持 IPSec / SSL 接入，适合安全移动访问场景 |

## **CPE类型（Customer Premise Equipment）**

- **二层方案（L2）**：客户自备设备。
- **三层方案（L3）**：
    - LL：客户自备。
    - IPSec：客户自备或使用 Fortinet。
    - SSL：使用 Fortinet Client 或 OpenVPN。
    - BMC/VM/ZEC：平台自带连接能力。

## **场景一：二层专线方案（Layer 2 Solution）**

**应用方向：**

- 企业数据中心 ↔ 海外办公室之间建立点到点二层专线；
- 游戏加速、直播（TikTok）等低延迟传输。

**连接结构：**

- 上海机房 ↔ 上海POP ↔ 东京 ↔ 西雅图 ↔ 圣何塞POP ↔ 硅谷办公室。
- 使用 Zenlayer SDN骨干网（含APG/NCP/PC-1North 海缆路由）。

**优势：**

- 丰富的节点与海缆资源；
- 专线级低延迟；
- 骨干带保护路径，自动切换防止中断。

**客户关注：**

- 接入方式（LL/XC）；
- 网络性能数据；
- 故障切换机制；
- 主动告警与运维。

**延伸应用：**

- **游戏加速器**：提供极低延迟及本地IP应对游戏锁区。
-  **TikTok直播**：关注BGP路由及本地IP可用性，确保推流稳定。

**适用场景：**

- **游戏**：办公网组网需求、游戏加速器
- **跨境电商：**Tiktok直播带货需求
- **SDWAN服务商/运营商**：纯转售最终客户需求、Tiktok直播带货需求
- **跨国企业、大型外企：**跨国分支机构组网

## **场景二：多云互联（Multi-Cloud Interconnection）**

**应用方向：**

- 企业多云架构（AWS/Azure/阿里云/GCP）间的私网互通；
- IDC ↔ 云、云 ↔ 云、办公网 ↔ 云。

**连接结构：**

- 双端（A端与B端）通过 LL/XC/DX 或 IPSec 接入 Zenlayer SDN；
- Zenlayer骨干完成全球互联。

**优势：**

- 丰富的云直连资源；
- 跨境/同城/跨区域灵活接入；
- 支持点到点、二层、三层等多样方案；
- 可扩展Virtual Edge方案（如虚拟PoP）。

**客户关注：**

- 云端与DC的接入方案；
- 性能监控与切换机制；
- 路径冗余与恢复时间；
- 主动运维与告警服务。

**典型连接模式：**

- Option 1：一条DX通向多个Region（集中出口DXG）；
- Option 2：多条DX分别直连多个Region（独立出口）。

**示例行业：**

- 游戏（边缘服 ↔ 云中心服）；
- 音视频RTC（跨境实时互动）。

**适用场景：**

- **游戏**：多云间业务数据传输、游戏加速、战斗服回源
- **音视频、RTC：**公有云的管理需求、跨区域实时互动卡顿
- **跨境电商：**第三方系统和主站的数据同步、边缘站点与主站的数据同步
- **SDWAN服务商/运营商**：纯转售最终客户需求、自身骨干网扩容
- **跨国企业、大型外企：**企业上云，资源管理、云间互通

## **场景三：全球组网（Enterprise WAN）**

**应用方向：**

- 跨国公司总部 ↔ 各地分支 ↔ 公有云；
- 同时优化 SaaS 访问和远程安全接入。

**连接结构：**

- 各地总部通过LL/IPSec接入 Zenlayer骨干；
- 用户、云平台通过SSL/DX接入；
- 提供MCE/SD-WAN/Fortinet方案。

**优势：**

- 全球范围的组网；
- 支持SASE、安全统一管理；
- 兼容第三方终端与SaaS平台；
- 可提供SaaS加速白名单服务。

**客户关注：**

- 全球覆盖；
- 稳定性与QoS；
- 企业级服务能力。

**适用场景：**

- **游戏**：办公网组网需求
- **跨国企业、大型外企：**跨国分支机构组网、海外应用、Saas加速

## **场景四：增强型跨国组网（Compliance WAN）**

**应用方向：**

- 面向中国区与海外互通的合规解决方案。
- 企业需满足监管要求（如“五真备案”）。

**连接结构：**

- 办公室通过LL/IPSec接入；
- 移动用户通过SSL加密接入；
- VM/ZEC中转至Global Internet或云资源。

**重点：**

- 提供行为管理与移动用户接入管控；
- 符合合规备案要求，保障合法跨境数据访问。

**适用场景：**

- **游戏**：办公网组网需求、海外游戏测试
- **跨国企业、大型外企：**跨国分支机构组网、海外应用、Saas加速

## **场景五：骨干网建设（Backbone Partnership）**

**目标客户：**

- 国内运营商；
- SD-WAN服务商；
- 渠道商与代理商。

**架构说明：**

- Zenlayer海外骨干 ↔ 客户国内骨干；
- NNI（Network to Network Interface）互联；
- 本地机房或公有云接入。

**特点：**

- 提供资源与方案合作；
- 可在香港或海外POP建立NNI以规避监管风险；
- 为合作伙伴客户提供TikTok/游戏加速服务；
- 支持BMC/VM节点直连。

## **案例一：游戏战斗服回源**

### **客户背景**

- 客户为游戏厂商，核心业务依托腾讯云部署。
- 区域服、中心服部署在腾讯云；战斗节点需在全球边缘节点覆盖。
- 部署城市包括：芝加哥、圣保罗、雅加达、墨尔本（战斗服）；
  
    弗吉尼亚、新加坡、法兰克福、孟买（中心服）。
    

### 客户需求

- 建立中心服与战斗服之间的专线骨干互联；
- 确保各区域节点玩家能就近接入；
- 提升腾讯云在部分地区节点覆盖不足时的连接性能与稳定性；
- 实现高可用架构，保证回源质量和可靠性。

### **解决方案**

- 区域服与中心服均部署于腾讯云；战斗节点接入 Zenlayer 边缘节点；
- 采用 **M 型号 BMC 进行本地化部署**，提升可用性；
- 双专线接入腾讯云，保证回源质量与高可用性；
- 通过 Zenlayer Backbone 形成中间层骨干连接。

### 实施架构

战斗服节点（芝加哥/圣保罗/雅加达/墨尔本）
│
│  专线骨干互联
▼
Zenlayer Backbone（区域节点）
│
│  接入腾讯云区域服（法兰克福/孟买/新加坡/弗吉尼亚）
▼
中心服 ↔ 战斗服回源

## **案例二：游戏多云互联**

### **客户背景**

- 客户为游戏公司，核心业务运行在新加坡阿里云和 AWS。
- 两朵云分属不同 Region，需进行跨云通信。

### 客户需求

- 实现新加坡阿里云到新加坡 AWS 的稳定互联；
- 支持双线接入，减少跨云延迟与路由跳转；
- AWS 传输数据需回传至阿里云进行分析。

### **解决方案**

| **模块** | **内容** |
| --- | --- |
| 网络接入 | 双线分别接入阿里云与 AWS，不同 Region |
| 中间互联 | 通过 Zenlayer Backbone（SIN1PE4 / SIN2PE4）实现跨云组网 |
| 路由协议 | BGP Peering，保持动态路由收敛 |
| 云端连接 | 阿里云：Express Connect（VBR+CEN+vRouter）AWS：DXGW / VGW |
| 中立互联 | 采用 ECX、Megaport 等互联平台打通云间通信 |

### 实施架构

阿里云（VPC + CEN + vRouter + VBR）
│  Express Connect
▼
Zenlayer Backbone（SIN1/SIN2 双节点）
│  DXGW + BGP Peering
▼
AWS（VGW）

## **案例三：电商管理及数据同步**

### 客户背景

- 客户为深圳一家做欧美母婴市场的跨境电商平台；

深圳 IDC ─── AWS 宁夏 ─── AWS 美西
（Full Mesh，双链路）

## **案例四：跨国企业组网（Global Enterprise WAN）**

### 客户背景

- 客户为跨国制造业或科研机构企业；
- 企业在中国多个城市及欧洲均设有生产基地和研发中心；
- 主要节点包括：
    - 中国：武汉（生态城、中交城、设计广场、工厂、数据中心）、杭州（阿里云、数据中心、分支机构）；
    - 欧洲：法国（支付、AWS、数据中心）、荷兰、德国、英国；
    - 其他：深圳科兴、伦敦数据中心。

### 客户需求

- 构建企业内部统一网络，连接总部、工厂、数据中心、云资源；
- 实现多地区、跨境低延迟通信；
- 支持 Internet 与专线混合接入模式；
- 实现全局安全访问与集中化管理。

### 解决方案

- 核心骨干采用 **Zenlayer SDN 网络**（节点：WUH、SHA、HGH、FR、LO、CAN、SZX、FRAP）；
- 安全体系由 **Fortigate VM + FortiAnalyzer + FortiManager** 组成；
- 接入方式分为 Internet 与专线（一期/二期）建设；
- 云资源包括 **阿里云与 AWS（法国）**；
- 各分支带宽 50–100M，核心节点最高至 1G。

### 实施架构

中国区（武汉/杭州/深圳等分支）
│ Internet + 专线（双路径）
▼
Zenlayer SDN 骨干网（多 POP：WUH、SHA、HGH、FR、LO、SZX）
│
│ Fortigate 安全接入
▼
欧洲区（法国、荷兰、德国、英国）+ 云资源（AWS、阿里云）
