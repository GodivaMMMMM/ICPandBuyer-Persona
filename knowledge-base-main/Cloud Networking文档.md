# Zenlayer云网络

## **当今企业网络需求的升级**

**核心痛点：**

- 传统网络连接方式（如运营商专线）已无法满足企业快速部署、多云接入和弹性扩展需求。
- 企业数字化转型驱动“应用分布全球化”，对网络性能、灵活性和可靠性要求大幅提高。

**典型需求：**

- 混合云、多云架构互联
- 按需使用、灵活计费
- 统一监控与告警
- 低成本试用、快速开通业务
- 性能可视化与智能调度

**总结一句话：**

> 企业网络连接正从“静态、固定带宽”向“云化、灵活、智能化”的 NaaS（Network-as-a-Service）模式升级。
> 

## **了解 Zenlayer 云网络**

**核心理念：**

Zenlayer 云网络（Zenlayer Cloud Networking） = **服务云化 + 网络资源**

- **服务云化**：
  
    按需提供多点互联服务，实现企业、云、SaaS、IDC 等之间的灵活连接。
    
- **网络资源**：
  
    覆盖全球的 PoP 点、骨干专线与云间直连资源，可接入公有云、私有云及边缘节点。
    
- **Zenlayer 云网络的价值：**
  
    支撑用户“按需构建网络”，实现 **数据中心与云之间的直达连接**，提供 **NaaS（Network as a Service）** 服务。
    

 **一句话概括：**

> Zenlayer 云网络让企业像使用云主机一样，快速创建并管理全球网络连接。
> 

## **连接云与数据中心**

Zenlayer 通过全球 SDN Backbone 实现与主流云厂商的高性能互联：

**支持的云厂商：**

AWS、Azure、Google Cloud、Alibaba Cloud、Tencent Cloud、Oracle、Huawei Cloud、UCloud 等。

**功能特性：**

- 支持 L2 二层透传
- 支持 L3 三层组网
- 支持 95 带宽灵活计费
- 支持链路冗余与保护
- 支持业务秒级切换

 **业务场景：**

- 企业总部与云互联
- 分支机构与数据中心互通
- 跨云互联（Cloud-to-Cloud）

 **一句话概括：**

> Zenlayer SDN Backbone 让企业在数分钟内即可完成“总部—分支—云”的全球高可靠互联。
> 

## **Cloud Networking 产品线**

Zenlayer 的云网络产品由五大核心模块构成，并支持云网络混合组网场景：

| 模块                                                         | 技术说明                                                     | 功能定位                                                     |
| --- | --- | --- |
| **Private Connect （二层专线打通，安全、稳定、透明的点对点网络）** | * Zenlayer 骨干网负责报文透明传输； * 客户侧可在 CE 设备上进行 BGP/IGP 协议交互； * 适用于需要高安全性、低时延、透明传输的场景。 | * 提供 **二层专线直连服务**，两端站点通过二层网络互通。 * 无需加密即可保障流量安全，支持协议透传（LLDP、ISIS、LACP、STP、MACSEC、E-LMI、PAUSE 等）。 * 支持 MTU > 9000，满足高吞吐业务需求。 |
| **Cloud Router （三层智能组网，助力企业快速构建全球 IP 通道）** | * Zenlayer 骨干 PE 与客户 CE 建立 BGP/静态路由；* 支持多个 CE 间的 IP 路由传递； * 适合多分支、多云、多数据中心之间的三层互联。 | * 提供 **三层组网服务**，实现多站点间的 IP 互通。 * 支持 Full-mesh / Hub-spoke / 自定义拓扑结构。 * 接入侧支持 BGP 或静态路由。 |
| **Cloud Connect （多云互联与云-DC 专线，打造一跳即达的高性能云间网络）** | * 物理冗余设计，保障高可靠性； * 可快速在多云间建立稳定、安全的互联。 | * 实现 **公有云与公有云、公有云与数据中心之间** 的高速专线互联； * 支持二层连接或三层组网； * 对接主流云厂商专线接口，如 AWS、Azure、GCP、Alibaba、Huawei、Tencent、Oracle、IBM 等。 |
| **Virtual Edge （软件定义边缘接入，让企业轻量、安全地融入全球云网络）** | * 企业办公点、远程员工、虚拟机（VM）通过 VPN/GRE 快速接入； * 支持与 SaaS（如 Zoom、Office 365、Salesforce）及主流云互通。 | * 适用于无法使用专线接入的“最后一公里”客户； * 通过 **Internet + IPSEC VPN / GRE Tunnel** 接入 Zenlayer 自研软件网关（MCE）或第三方 vPOP； * 提高部署灵活性、降低成本。 |
| **IPLC （极致性能的国际传输专线，提供物理级稳定与可视化监控）** | * ZenConsole提供流量监控、告警日志、性能分析等可视化管理； * 支持多样化 SLA 与实时带宽监测。 | * 提供 **L1 点对点物理专线**； * 特性包括带宽独享、超大容量、完全透明传输； * 适用于跨国高带宽、低时延业务场景。 |
| **云网络混合组网场景 （企业总部、分支、数据中心与多云的统一互联架构）** | * 核心由 Zenlayer Backbone 支撑； * Local Loop / XC 实现数据中心或办公点接入； * Virtual Edge Gateway 提供互联网侧弹性连接； * 支持 AWS、Azure、GCP、阿里云、腾讯云、华为云等主流公有云。 |企业、私有云、公有云、数据中心多点互联；企业需连接国内外不同云区域；不同云厂商/区域之间的互通；最后一公里通过 Internet 接入的用户。|
**一句话总结：**

> Zenlayer Cloud Networking 提供从专线互联到云端组网的全栈产品体系，满足不同企业规模与场景的网络需求。



# 云网络产品特色

- **云网络 vs 传统网络连接 —— 敏捷、灵活、智能的新时代网络**

| 对比维度 | 云网络 | 传统网络连接 |
| --- | --- | --- |
| **部署周期** | 秒级即时配置 | 部署时间长，往往需数月 |
| **弹性 / 可扩展性** | 按需扩展或收缩带宽 | 带宽无法轻易更改 |
| **价格** | 按需使用，按量计费 | 价格固定且非常昂贵 |
| **租期** | 按月租用 | 按年签署合同 |
| **云网络优化** | 针对多云网络进行优化 | 通常不提供与公有云的专用连接 |
| **可用性** | Zenlayer 一站式全球连接 | 跨地区、跨运营商的网络连接非常复杂 |
| **易用性 / 可视化** | 可视化平台、自助服务，提供流量、时延、丢包、抖动、端口状态、协议状态等详细信息 | 邮件、电话提供技术支持 |
| **维护便捷性** | 支持按需自定义告警规则（如流量、时延、丢包告警） | 以维护通知为主 |

 **一句话总结：**

> Zenlayer 云网络以 SDN 架构实现全球快速部署、灵活扩容、按需计费和智能可视化运维，是传统专线的下一代替代方案。
> 

- **高性能网络 ——专属骨干，保障稳定与低时延**
    - **超低延迟**：高峰期延迟与抖动降低约 40%，独享高速通道。
    - **高可用性**：零丢包设计，核心路由节点具备硬件冗余，线路全链路保护。

**一句话总结：**

> Zenlayer 专属骨干网络以高可靠架构和智能调度算法，确保业务“高速稳定不掉线”。
> 

**广泛互联 —— 全球覆盖 + 多云直连**

- 覆盖全球 200+ 节点，提供极致交付速度与最佳网络体验；
- 支持混合云、多云架构快速部署；
- 直连主流云与 SaaS（AWS、Azure、Google Cloud、Alibaba Cloud、Tencent Cloud、Huawei、Oracle 等）；
- 与全球顶级互联伙伴（Megaport、Equinix 等）深度对接。

**一句话总结：**

> Zenlayer 通过全球节点和多云互联生态，构建企业一跳直达的全球云网络。
> 

- **弹性计费 —— 灵活的用量与成本管理**

**支持多种计费模式：**

- 固定带宽模式
- 保底突发模式
- 分时段差异计费（如白天/夜间）
- 按流量包计算

**带宽调节：**

- 按需增减（1 Mbps–10 Gbps 可选）
- 支持突发带宽模式

 **一句话总结：**

> 灵活的带宽与计费策略帮助企业实现“按需用网、精准控费”。
> 

- **全网可视 —— 端到端的网络可观测性**
    - **业务状态可视**：秒级监测流量、时延、端口与协议状态。
    - **线路状态可视**：全链路的流量、丢包、延迟等数据透明展示。
    - **智能告警规则**：客户可自定义阈值（如流量 / 延迟 / 丢包）并自动触发告警。

 **一句话总结：**

> Zenlayer 提供全网级可视化能力，让企业随时掌握网络健康状况与性能趋势。
>
