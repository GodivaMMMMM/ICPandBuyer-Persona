# 端口介绍

- 端口是 Zenlayer 基于 SDN 架构推出的创新型网络销售与服务模式。它区别于传统的“电路逐条销售”，通过 **一次端口接入，即可多次复用、快速开通全网服务**，帮助客户以更灵活、更低成本的方式构建全球互联网络。
- **传统模式 vs 端口模式**

| 对比项 | **传统电路模式** | **端口方案模式** |
| --- | --- | --- |
| **销售方式** | 逐条销售，每开通一条线路需单独交付 | 一次端口，多场景复用 |
| **部署效率** | 流程繁琐、交付周期长 | 即开即用，自助开通 |
| **成本结构** | 固定高成本 | 按需计费，成本更优 |
| **客户体验** | 可见性差、依赖人工配置 | 自助管理、灵活可视 |
| **用户粘性** | 弱 | 高（端口绑定客户账户，随时扩展新业务） |
- **一个端口，一点接入，全网通达，快速开通，灵活计费：**通过客户侧的一个端口（Port），连接至 Zenlayer SDN 全球骨干网，客户可以在同一个物理或虚拟端口上，按需开通多种网络服务，包括公有云互联、多云交换、数据中心互联、弹性计算节点、优质互联网带宽
- 客户无需反复购买多条专线或独立资源，只需接入一个端口，即可获得对全球骨干网资源的访问权限。Zenlayer 的自助平台（ZenConsole）可实现实时开通、按天/按月计费、带宽弹性调整及业务可视化。

# 什么样的客户适用端口

| 客户类型 | 场景说明 | 端口模式价值 |
| --- | --- | --- |
| **MSP（合作伙伴）** | 自身已有骨干或客户网络，需借助 Zenlayer 网络拓展到更多节点、地区或公有云场景 | 通过端口快速连接 Zenlayer 全球 PoP，实现多客户共享资源、网络弹性扩展 |
| **公有云厂商 / 云服务提供商** | 需打通不同类型公有云（多云互联） | 端口可作为云与云间的中立交互点，简化多云对接，提高互联效率 |
| **企业用户** | 从自身 IDC 或办公室网络出发，希望快速实现全球访问（云、IDC、SaaS） | 购买一个端口即可连接 Zenlayer 网络，按需开通至各目标节点（Cloud/DC/BMC/IPT） |
| **现有客户（已有骨干网络）** | 已拥有自建专线或主干资源，但需临时扩展或增强可靠性 | 按需购买 Zenlayer 资源，作为备份或冗余线路，增强网络健壮性和可扩展性 |

# 端口模式的好处

- 效率高，客户可自服务下单，无需麻烦销售、PM、售前参与
- 自主性高，用户可通过已购买的端口，自助查看其他资源服务，并可快速开通新的业务服务
- 用户黏性高，给用户提供好的体验，并且灵活自助，性价比高

# 端口网络应用场景

**场景一：网络延展 / 即开即用**

- 客户已有自己的基础网络，希望通过购买 Zenlayer 端口，延展至其他城市或云节点。
一旦接入 Zenlayer Port，即可通过 SDN Backbone 快速打通全球场景
- 核心价值：快速延展 / 灵活计费 / 即开即用 / 支持多方向接入

**场景二：资源备份 / 按需扩展**

- 针对已有自建网络的客户，可通过购买 Zenlayer 端口获得“**备份链路 + 按需扩容**”能力。
- 核心价值：
    - 网络冗余，提升可靠性
    - 按需付费，灵活扩展
    - 快速交付，支撑高峰或临时需求
